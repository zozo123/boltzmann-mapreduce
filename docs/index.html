Why this is valuable

An honest interface for evidence-aware reduction.

The reducer carries evidence strength, declared identity, lineage, and disagreement from worker output to pooled result.

01

Evidence earns weight

Total precision Pₖ = nₖJₖ determines each estimate's contribution to the pooled result.

02

Merging scales as a tree

The numeric canonical state is associative and commutative. Provenance is aggregated separately with explicit field semantics.

03

Conflict stays visible

The product normalizer contains a disagreement term. Confident workers that point in different directions leave an explicit audit signal.

The scientific boundary

Separate workers still require independent evidence.

Multiplication applies when workers estimate the same target from independent, non-overlapping evidence. The evidence design carries this scientific burden.

✓

Use it when

Worker summaries have a defensible uncertainty model and genuinely distinct evidence.

  • One shared parameter and compatible units
  • Disjoint evidence or explicitly modeled dependence
  • Valid Gaussian or local-asymptotic approximation
×

Requires a different model

Correlated agents, adversarial workers, and different local truths require explicit dependence or robustness models.

  • Self-reported model confidence
  • An automatic Byzantine defense
  • Proof that separate sandboxes are independent
Core requirementShared prompts, models, repositories, tests, or fork ancestors create common-mode error. Provenance supplies the audit trail; correlation modeling remains a separate requirement.

Small enough to inspect. Complete enough to rerun.
